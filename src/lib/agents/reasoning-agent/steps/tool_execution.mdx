If the LLM outputted an `Action` in the previous step, the application software intercepts this. 

The LLM *does not* execute code itself. The application takes the `Action Input`, runs the actual python function or API call (like fetching a weather API), captures the result, and formats it as an `Observation`. This observation is then appended to the message history, triggering the start of a new `llm_cycle_prompt`.
